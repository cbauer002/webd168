Movement

Moving a player is one of the most important parts of making a game. From adventure to fighting games, basic 4 way movements (up, down, left, right) for 2d games create a bigger world than just what is shown on the screen. In example 1, we can see the player move left, right and jumping. Much like how html has event listeners for mouse clicks, there is a process that waits for keyboard clicks. If the player presses "A" then it will make the character move to the left. When a space bar key is detected, then it makes the player jump.

var direction := Input.get_axis("move_left", "move_right")

This line of code is the one that detects the "A" and "S" key being pressed. Using that, we are able to tell the code to move the character left or right. It is possible to reverse the code where a player can go left when pressing right. These are how inverted controls are made.

if Input.is_action_just_pressed("jump")

This line detects if the player has pressed the space bar key which will allow the player to jump.

Colliders

When a character has reached a door or wall, how does the computer know when not to allow a player to pass through a wall or door? Colliders is one of the ground works to creating a video game. Without colliders, characters will just fall on the screen forever. Creating a ground with a collider so the character can walk and jump.

In example 1, you can see that the level is normal without any highlights. In example 3, I have turned on the feature to show colliders. You can now see that the ground is highlighted in a light green. This is the collider that allows the player to stand on top. You can see that the 2 trees in the level are not highlighted green. The player is able to pass through the trees, but not the ground or the placed stones blocking the bridge.

Coin

When the colliders hit each other, how does it know when it has hit an enemy or a coin? When objects are created, they are given a name so that when the colliders are detected, it can read the name of the object and executes the code that matches the name. So if the character runs into an enemy, the code will decrease health until there is no more health. At that point the character dies and gets reset. When the character runs into a coin, the code will add plus one to their coin count and make the coin disappear.

This block of code belongs to the coin. When the player collider touches the coin collider, the code will add a point to the counter and then play a tone that tells the player that a coin was picked up.
This block of code belongs to enemies. Whenever the player touches an enemy, the player will die and then will re-spawn back in the level or have the level reset completely.
